# 第 7 课：什么是 Agent

版本日期：2026-05-29  
阅读对象：全体同事，重点适合 AI 种子用户、产品、运营、项目管理和技术协作人员  
预计阅读时间：12-15 分钟  
课程目标：帮助大家用非技术语言理解 Agent，知道它和聊天助手、Workflow 的区别，并学会写一个基础 Agent 委托。

## 一、先记住这三句话

1. 聊天助手主要回答问题，Agent 更像围绕目标做事的 AI 助手。
2. Agent 通常由目标、工具、上下文和反馈循环组成。
3. Agent 能力更强，权限、边界和验收要求也必须更清楚。

如果你只记住本课一个观点，请记住：

> Agent 不是“更聪明的聊天框”，而是一个可能会使用工具、读取资料、分步骤执行任务的系统。

## 二、从“问 AI”到“委托 AI 做事”

普通聊天助手的常见使用方式是：

```text
请帮我总结这段文字。
```

它通常会根据你提供的内容直接回答。

Agent 的使用方式更像委托任务：

```text
请阅读这些资料，先列出计划，再整理竞品对比表，发现信息缺口后向我提问，最后输出一份内部讨论用简报。
```

这个任务不只是一次回答，而是包含多个步骤：

1. 阅读资料。
2. 制定计划。
3. 提取信息。
4. 发现缺口。
5. 向人提问。
6. 输出简报。

这就是 Agent 的典型特征。

## 三、聊天助手、Workflow、Agent 的区别

| 类型 | 通俗解释 | 适合任务 | 风险 |
| --- | --- | --- | --- |
| 聊天助手 | 你问一句，它答一句 | 起草、改写、总结、解释 | 可能泛泛而谈或出错 |
| Workflow | 固定步骤的 AI 流程 | 标准纪要、周报、工单分类 | 流程设计错会稳定地产生错结果 |
| Agent | 围绕目标使用工具、观察结果、继续调整 | 开放式研究、跨文件整理、资料分析、代码或文档修改 | 权限、误操作、成本、过度信任 |

简单判断：

- 单次生成：用聊天助手。
- 固定步骤、重复出现：用 Workflow。
- 开放式、多步骤、需要动态判断：考虑 Agent。

## 四、Agent 的四个组成

### 1. 目标

目标是 Agent 要完成的最终结果。

示例：

- 整理一份竞品简报。
- 检查一组文档中的风险。
- 生成一份培训材料。
- 根据多份会议纪要整理项目行动项。

目标必须清楚，否则 Agent 很容易跑偏。

### 2. 工具

工具是 Agent 能调用的能力。

可能包括：

- 搜索网页。
- 阅读文件。
- 写入文档。
- 分析表格。
- 调用日历。
- 查询知识库。
- 执行代码。
- 连接业务系统。

工具越多，能力越强，风险也越高。

### 3. 上下文

上下文是 Agent 完成任务需要知道的背景和规则。

包括：

- 公司背景。
- 项目背景。
- 目标读者。
- 输出模板。
- 禁止事项。
- 数据安全要求。
- 业务判断标准。

### 4. 反馈循环

反馈循环是 Agent 做完一步后观察结果，再决定下一步。

例如：

1. 先阅读资料。
2. 发现资料缺少价格信息。
3. 向用户提问。
4. 根据补充信息更新对比表。
5. 输出最终简报。

这也是 Agent 和普通聊天助手的重要区别。

## 五、什么时候适合用 Agent

适合 Agent 的任务通常具备这些特点：

1. 需要多步处理。
2. 需要读取多份资料。
3. 需要在过程中判断下一步。
4. 需要整理复杂信息。
5. 需要产出结构化交付物。
6. 可以设置清晰边界和验收标准。

示例：

- 整理 20 份公开资料，形成行业研究初稿。
- 跨多个项目文档找风险并生成清单。
- 从多次会议纪要中提取未完成行动项。
- 根据一组脱敏客户反馈整理问题分类和改进建议。

## 六、什么时候不适合用 Agent

以下情况不建议直接用 Agent 自动处理：

1. 单次聊天就能完成的简单任务。
2. 涉及付款、删除、提交、发送、改权限的动作。
3. 涉及法律、财务、人事、医疗、安全的正式判断。
4. 输入材料包含大量敏感信息且没有授权环境。
5. 任务目标不清楚，无法验收。

一个重要原则：

> 能用聊天助手解决，就不要复杂化；能用固定 Workflow 解决，就不必上 Agent；只有任务复杂、开放、需要多步探索时，再考虑 Agent。

## 七、完整示例：竞品简报任务如何 Agent 化

### 1. 普通聊天式 Prompt

```markdown
帮我写一份竞品简报。
```

问题：

1. 没有说明竞品是谁。
2. 没有说明资料来源。
3. 没有说明输出格式。
4. 没有限制 AI 不要编造数据。
5. 没有说明这是内部讨论还是对外材料。

### 2. Agent 委托 Prompt

```markdown
目标：
请帮我整理一份竞品简报初稿，供内部讨论使用。

可用资料：
1. 我提供的竞品官网公开介绍。
2. 我提供的产品对比表。
3. 不允许使用未提供来源的具体数字。

允许做的事：
- 阅读我提供的资料。
- 提取产品定位、目标客户、核心功能、价格线索。
- 整理差异和待确认问题。

不允许做的事：
- 不要编造市场份额、收入、客户数量。
- 不要输出任何对外宣传语。
- 不要访问或处理公司机密资料。

过程要求：
1. 先列出你的工作计划。
2. 如果资料不足，先问我问题。
3. 输出中区分“已知事实”和“推测/待确认”。

交付物：
用 Markdown 输出，包含摘要、对比表、机会点、风险、待确认问题。
```

### 3. 为什么第二个更适合 Agent

它不仅说明了要做什么，还说明了：

1. 可用资料是什么。
2. 允许做什么。
3. 不允许做什么。
4. 过程中如何汇报。
5. 事实和推测如何区分。
6. 最终交付物长什么样。

这就是 Agent 委托与普通聊天提问的区别。

## 八、Agent 委托模板

可以保存下面模板：

```markdown
目标：
{希望完成的最终结果}

可用资料：
{文件、链接、文本、表格、上下文}

允许做的事：
{读取、总结、分类、生成草稿、提出问题、运行检查}

不允许做的事：
{发送、删除、付款、正式提交、处理敏感数据}

过程要求：
1. 先复述目标和计划。
2. 信息不足先提问。
3. 分阶段汇报结果。
4. 标注事实、推测和待确认。

交付物：
{文档、表格、清单、邮件草稿、PPT大纲}

验收标准：
{准确、完整、可执行、安全、格式符合要求}
```

## 九、任务分流练习

请看下面任务更适合哪种方式：

| 任务 | 推荐方式 | 原因 |
| --- | --- | --- |
| 改写一段通知 | 聊天助手 | 单次生成即可 |
| 每周固定把会议记录变成行动项 | Workflow | 步骤固定，可模板化 |
| 整理 20 份公开资料形成行业研究初稿 | Agent | 多资料、多步骤、需补缺口 |
| 让 AI 自动批准报销 | 不建议自动执行 | 高风险审批动作 |
| 把客户反馈分成 5 类 | Workflow 或聊天助手 | 看规模和频率 |
| 生成一个活动方案第一稿 | 聊天助手 | 初稿生成即可 |
| 跨多个文档找项目风险并生成清单 | Agent | 需要读多份资料和综合判断 |

## 十、Agent 使用前检查清单

| 检查项 | 是/否 |
| --- | --- |
| 目标是否清楚 |
| 可用资料是否明确 |
| 是否排除了敏感信息 |
| 允许做的事是否写清 |
| 禁止做的事是否写清 |
| 是否要求先列计划 |
| 是否设置人工审批点 |
| 是否有验收标准 |

## 十一、本课自测

### 题目 1

Agent 和聊天助手最大的区别是什么？

参考答案：Agent 可以围绕目标使用工具、分步骤执行任务，并根据中间结果调整下一步。

### 题目 2

为什么不是所有任务都要用 Agent？

参考答案：Agent 更复杂，成本和风险更高。简单任务用聊天助手或 Workflow 更合适。

### 题目 3

Agent 委托里为什么要写“不允许做的事”？

参考答案：为了控制权限和风险，避免误操作或处理不该处理的数据。

### 题目 4

哪些动作必须设置人工审批？

参考答案：发送、删除、付款、提交正式系统、修改权限、对外承诺等。

## 十三、本课总结

Agent 是 AI 从“回答问题”走向“执行任务”的重要形态。

你需要记住：

1. 聊天助手适合单次问答和生成。
2. Workflow 适合固定步骤和重复任务。
3. Agent 适合开放、多步、需要动态判断的任务。
4. Agent 必须写清目标、资料、权限、过程和验收。
5. 能执行不代表可以自动执行，高风险动作必须人工审批。
