# 第 4 课：六要素提示词模板

版本日期：2026-05-29  
阅读对象：公司全体同事  
预计阅读时间：12-15 分钟  
课程目标：帮助大家掌握一套可复制的 Prompt 写法，用于邮件、纪要、方案、总结、分析和检查等常见办公任务。

## 一、先记住这三句话

1. 好 Prompt 不靠玄学，靠把任务讲完整。
2. 六要素是：角色、目标、背景、资料、输出、限制。
3. 复杂任务最好再加一项：验收标准。

如果你只记住本课一个观点，请记住：

> 六要素模板就是给 AI 的工作单。工作单越清楚，输出越稳定。

## 二、为什么需要模板

很多同事写 Prompt 时会遇到这些问题：

1. AI 输出太泛。
2. AI 语气不符合场景。
3. AI 输出格式不能直接用。
4. AI 添加了原文没有的信息。
5. AI 不知道哪些地方不能碰。

这些问题大多数不是因为 AI “不会”，而是因为任务交代得不够完整。

六要素模板可以帮助你稳定地讲清楚任务。

## 三、六要素总览

| 要素 | 含义 | 示例 |
| --- | --- | --- |
| 角色 | 让 AI 站在什么视角处理 | 你是一名内部培训讲师 |
| 目标 | 要完成什么具体任务 | 生成一份 15 分钟课程大纲 |
| 背景 | 为什么做、给谁看、用于什么场景 | 学员是非技术员工 |
| 资料 | AI 可以依据什么内容 | 以下是会议记录 |
| 输出 | 用什么格式交付 | 表格、清单、邮件、Markdown |
| 限制 | 不能做什么、注意什么 | 不要编造事实，不确定请标注 |

可以加第七项：验收标准。

验收标准的作用是告诉 AI 什么样的结果才算可用。例如：

```text
领导 1 分钟内能看懂项目状态，并知道哪里需要支持。
```

## 四、逐项理解六要素

### 1. 角色

角色决定 AI 用什么视角处理任务。

示例：

- 你是一名项目助理。
- 你是一名内部培训讲师。
- 你是一名客服质检专员。
- 你是一名销售运营分析助理。

注意：角色只是处理视角，不代表 AI 真的具备专业资格。法律、财务、人事等专业结论仍需专业人员审核。

### 2. 目标

目标要说明你最终想得到什么。

模糊目标：

```text
帮我处理一下。
```

清楚目标：

```text
请把下面的工作记录整理成一份周报，突出本周进展、下周计划、风险和需要支持。
```

### 3. 背景

背景帮助 AI 判断深度、语气和重点。

示例：

```text
这份周报发给直属领导，目的是让领导快速了解项目状态，不需要展开技术细节。
```

### 4. 资料

资料是 AI 可以依据的内容。资料越明确，AI 越不容易泛泛而谈。

注意：

1. 不要粘贴敏感信息。
2. 指令和资料最好分开。
3. 缺失信息要让 AI 标注“需确认”。

### 5. 输出

输出决定结果能不能直接使用。

你可以要求 AI 输出：

- Markdown 表格。
- 邮件正文。
- 三段式总结。
- 行动项清单。
- PPT 大纲。
- FAQ。
- 检查表。

### 6. 限制

限制可以减少跑偏和风险。

常见限制：

1. 不要添加原文没有的信息。
2. 不确定内容请标注“需确认”。
3. 不要输出夸张宣传语。
4. 不要包含敏感信息。
5. 不要给出正式法律、财务、人事结论。

## 五、六要素通用模板

你可以保存下面这段，以后直接替换花括号内容。

```markdown
你现在扮演：{角色}

我要完成的任务：{目标}

背景信息：
{业务背景、目标读者、使用场景}

输入资料：
{粘贴材料，或说明资料来源。不要粘贴敏感信息}

请输出：
{格式、结构、字数、语言风格}

限制条件：
{不要编造、需标注不确定项、敏感信息处理、语气要求}

验收标准：
{我会如何判断这个结果能不能用}
```

## 六、完整示例：把工作记录整理成周报

### 1. 原始记录

```text
本周完成 AI 培训大纲第一版，收集了 HR 和运营部门的需求，发现大家最关心 Prompt、会议纪要、数据安全。下周计划做 8 节入门课的课件。风险是讲师时间不够，资料还需要法务确认。
```

### 2. 六要素 Prompt

```markdown
你现在扮演：一名项目助理。

我要完成的任务：把下面的工作记录整理成一份周报。

背景信息：
周报发给直属领导，目标是让领导快速了解进展、风险和下周计划。

输入资料：
本周完成 AI 培训大纲第一版，收集了 HR 和运营部门的需求，发现大家最关心 Prompt、会议纪要、数据安全。下周计划做 8 节入门课的课件。风险是讲师时间不够，资料还需要法务确认。

请输出：
用 Markdown 表格输出，包含：本周进展、关键发现、下周计划、风险与需要支持。

限制条件：
- 不要添加原文没有的数据。
- 语气简洁、正式。
- 不确定内容标注“需确认”。

验收标准：
领导 1 分钟内能看懂项目状态，并知道哪里需要支持。
```

### 3. 可能得到的输出

| 模块 | 内容 |
| --- | --- |
| 本周进展 | 完成 AI 培训大纲第一版；收集 HR 和运营部门需求 |
| 关键发现 | 各部门最关注 Prompt、会议纪要、数据安全 |
| 下周计划 | 制作 8 节入门课课件 |
| 风险与需要支持 | 讲师时间不足；资料需法务确认 |

### 4. 为什么这个 Prompt 稳定

1. 角色明确：项目助理。
2. 目标明确：整理周报。
3. 背景明确：给直属领导看。
4. 资料明确：只基于工作记录。
5. 输出明确：Markdown 表格。
6. 限制明确：不加数据、不确定标注。
7. 验收明确：领导 1 分钟能看懂。

## 七、常见场景模板

### 1. 邮件改写

```markdown
你现在扮演：一名商务沟通助理。

我要完成的任务：把下面这段内容改写成一封正式邮件。

背景信息：
邮件发给合作方，目的是确认会议时间和需要对方准备的材料。

输入资料：
{粘贴不含敏感信息的草稿}

请输出：
邮件标题和邮件正文。

限制条件：
- 语气礼貌、清晰、不过度承诺。
- 不要添加原文没有的信息。
- 缺少时间、地点或材料要求时标注“需确认”。

验收标准：
对方读完能明确知道会议安排和下一步动作。
```

### 2. 会议纪要

```markdown
你现在扮演：一名会议记录助理。

我要完成的任务：把下面的会议记录整理成会议纪要。

背景信息：
纪要发给参会人，用于会后跟进行动项。

输入资料：
{粘贴脱敏会议记录}

请输出：
1. 会议摘要。
2. 已确认决策。
3. 行动项表格：任务、负责人、截止时间、备注。
4. 风险和待确认事项。

限制条件：
- 负责人和截止时间缺失时写“需确认”。
- 不要把讨论意见写成已确认决策。

验收标准：
每个行动项都能被负责人跟进。
```

### 3. 文案检查

```markdown
你现在扮演：一名文案审校助理。

我要完成的任务：检查下面这段文案是否清晰、准确、得体。

背景信息：
文案将用于内部通知。

输入资料：
{粘贴文案}

请输出：
问题 | 风险 | 修改建议 | 修改后示例

限制条件：
- 不要改变事实。
- 不要添加额外承诺。
- 不确定内容标注“需确认”。

验收标准：
修改后内容清楚、礼貌、无明显遗漏。
```

## 八、Prompt 自评清单

写完 Prompt 后，请检查：

| 检查项 | 是/否 |
| --- | --- |
| 是否写了角色 |
| 是否写了具体目标 |
| 是否说明了背景和读者 |
| 是否提供了必要资料 |
| 是否指定输出格式 |
| 是否写了限制条件 |
| 是否说明不确定时怎么处理 |
| 是否有验收标准 |
| 是否避免输入敏感信息 |

## 九、本课自测

### 题目 1

六要素是哪六个？

标准答案：角色、目标、背景、资料、输出、限制。

### 题目 2

为什么要指定输出格式？

参考答案：指定输出格式可以让结果更接近可直接使用，也便于复制、检查和复用。

### 题目 3

如果资料里没有某个数据，应该要求 AI 怎么处理？

参考答案：不要编造，标注“不确定”或“需确认”。

### 题目 4

角色设定是否意味着 AI 真的具备专业资格？

标准答案：不是。角色只是一种处理视角，专业结论仍需专业人员审核。

## 十一、本课总结

六要素模板是最适合全员入门的 Prompt 写法。

你需要记住：

1. 角色决定处理视角。
2. 目标决定要完成什么。
3. 背景决定深度和语气。
4. 资料决定依据。
5. 输出决定结果形态。
6. 限制决定安全边界。
7. 验收标准决定是否可用。
