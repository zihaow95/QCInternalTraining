# 第 2 课：Prompt 是工作委托，不是搜索框

版本日期：2026-05-29  
阅读对象：公司全体同事  
预计阅读时间：12-15 分钟  
课程目标：帮助大家从“随便问一句 AI”升级为“像委托同事一样委托 AI”，让 AI 输出更清楚、更可控、更接近可直接使用。

## 一、先记住这三句话

1. 搜索框是找资料，Prompt 是交代工作。
2. AI 不会自动知道你的真实业务场景，需要你把目标、背景、资料、输出和限制说清楚。
3. 信息不足时，不要让 AI 猜，应该让它先提问或标注“需确认”。

如果你只记住本课一个观点，请记住：

> 写 Prompt 的本质不是“会提问”，而是“会清楚地布置一项工作”。

## 二、为什么不能只把 AI 当搜索框

很多人第一次用 AI，会像使用搜索引擎一样输入几个关键词。例如：

```text
AI 培训方案
```

这样的输入不是完全不能用，但它有明显问题：

1. AI 不知道方案给谁看。
2. AI 不知道培训对象是不是技术人员。
3. AI 不知道培训时长。
4. AI 不知道要输出大纲、讲稿、PPT 还是活动方案。
5. AI 不知道公司有哪些安全边界。

因此，它很可能输出一份泛泛而谈的内容，看起来完整，但很难直接拿来用。

更好的方式，是把 AI 当成一个需要接收任务说明的工作助理。你要像布置工作一样，告诉它任务目标、背景资料、输出格式、限制条件和判断标准。

## 三、搜索式提问和委托式 Prompt 的区别

| 维度 | 搜索式提问 | 委托式 Prompt |
| --- | --- | --- |
| 典型写法 | AI 培训方案 | 请为非技术员工设计一节 15 分钟 AI 入门课 |
| 目标 | 找一些相关信息 | 完成一个具体产出 |
| 背景 | 通常缺失 | 说明受众、场景、限制 |
| 输出 | 结果不可控 | 指定格式、长度、语气 |
| 验收 | 很难判断好坏 | 可以按标准检查 |

举例来说：

```text
帮我写个通知。
```

这句话很模糊。AI 不知道通知对象是谁、通知什么事、什么时候发生、要不要报名、语气要正式还是轻松。

更好的写法是：

```markdown
请帮我写一段企业微信群通知。

背景：
公司下周五下午 2 点在 5 楼培训室举办 AI 内部分享会，主题是如何用 AI 提升日常办公效率。各部门至少安排 1 名同事参加，需要在本周三前报名。

请输出：
100 字以内的群消息。

限制：
- 语气清晰、礼貌、直接。
- 不要添加原文没有的信息。
- 如果缺少报名方式，请标注“报名方式需确认”。
```

第二种写法更像一份工作委托，所以 AI 更容易输出可用结果。

## 四、好 Prompt 要回答 5 个问题

写 Prompt 时，可以先问自己 5 个问题：

1. 我要 AI 完成什么任务？
2. 这项任务的背景是什么？
3. AI 应该依据哪些资料？
4. 我希望它用什么格式输出？
5. 有哪些事情不能做，或者必须注意？

如果这 5 个问题都说清楚了，Prompt 通常就不会太差。

## 五、好 Prompt 的 5 个组成部分

### 1. 任务

任务是你希望 AI 做什么。任务越具体，输出越可控。

模糊写法：

```text
帮我总结一下。
```

更好写法：

```text
请把下面的会议记录总结成 5 条要点，并单独列出决策、行动项和待确认问题。
```

### 2. 背景

背景告诉 AI 为什么做、给谁看、用于什么场景。

示例：

```text
这份内容将发给部门负责人，目的是让他们快速了解项目进展和需要协调的风险。
```

没有背景时，AI 可能输出得太浅、太长、太正式或不符合对象。

### 3. 资料

资料是 AI 可以依据的内容。资料可以是一段文字、会议记录、表格、邮件草稿、公开链接摘要等。

需要注意：

1. 不要粘贴客户、合同、财务、人事、隐私等敏感信息到未批准的公开 AI 工具。
2. 如果资料不完整，要告诉 AI “缺失信息请标注需确认”。
3. 不要让 AI 根据没有提供的资料编事实。

### 4. 输出

输出是你希望结果长什么样。可以指定：

- 段落。
- 表格。
- 清单。
- 邮件。
- Markdown 文档。
- PPT 大纲。
- 行动项列表。

示例：

```text
请用 Markdown 表格输出，列为：任务、负责人、截止时间、风险、待确认事项。
```

### 5. 限制

限制是告诉 AI 不能做什么，以及遇到不确定情况怎么处理。

常用限制包括：

1. 不要添加原文没有的信息。
2. 不确定请标注“需确认”。
3. 不要使用夸张营销语。
4. 不要输出法律、财务、人事等正式结论。
5. 不要包含客户、员工或合同敏感信息。

## 六、完整示例：把模糊需求改成委托式 Prompt

### 1. 模糊需求

```text
帮我写个 AI 培训方案。
```

这个 Prompt 的问题是：

1. 不知道培训对象。
2. 不知道培训时长。
3. 不知道培训目标。
4. 不知道输出格式。
5. 不知道有哪些安全要求。

### 2. 改写后的 Prompt

```markdown
请为一家非技术员工占多数的公司设计一节 15 分钟 AI 入门培训课。

背景：
- 学员平时用过豆包、元宝等聊天式 AI 工具。
- 学员不懂编程。
- 培训目标是让他们知道 AI 能帮什么、不能帮什么，并能完成一次低风险跟练。

请输出：
1. 课程标题。
2. 3 个学习目标。
3. 15 分钟时间安排。
4. 现场演示 Prompt。
5. 课后作业。

限制：
- 用非技术语言。
- 不要夸大 AI 能力。
- 涉及客户、合同、人事、财务数据时，要提醒不能直接粘贴到公开 AI 工具。
- 如果信息不足，请先问我 3 个关键问题，不要直接编。
```

### 3. 为什么这个 Prompt 更好

它清楚说明了：

1. 任务：设计一节 AI 入门培训课。
2. 受众：非技术员工。
3. 背景：用过聊天式 AI，但不懂编程。
4. 输出：标题、目标、时间安排、演示、作业。
5. 限制：非技术语言、安全提醒、不足时先提问。

这就是“工作委托”的写法。

## 七、信息不足时，让 AI 先问问题

很多时候，我们自己一开始也没有想清楚任务。此时不要急着让 AI 生成完整答案，可以先让它帮你澄清需求。

可复制 Prompt：

```markdown
我想完成下面这个任务，但目前信息还不完整。

任务：
{简单描述你的任务}

请先不要直接生成结果。请先问我 3-5 个最关键的问题，帮助我补充背景、资料、输出格式和限制条件。问题要简洁，按重要性排序。
```

适合使用这个 Prompt 的场景：

- 写方案前还没想清楚目标。
- 做汇报前不知道受众关心什么。
- 写通知前缺少时间、地点或报名方式。
- 做总结前不知道要总结给谁看。
- 让 AI 帮忙分析问题前，资料还不完整。

## 八、通用委托式 Prompt 模板

你可以保存下面这个模板，日常工作中直接替换花括号内容。

```markdown
请帮我完成：{具体任务}

背景：
{为什么要做、给谁看、使用场景}

资料来源：
{粘贴材料，或说明材料范围。不要粘贴敏感信息}

请输出：
{格式、长度、结构、语气}

限制：
1. 不要添加我没有提供的事实。
2. 不确定内容请标注“需确认”。
3. 不要输出超出资料范围的结论。
4. 如涉及敏感信息、对外承诺或高风险判断，请提醒我人工审核。

如果信息不足，请先问我 3 个关键问题，不要直接生成。
```

## 九、常见工作场景示例

### 1. 写通知

```markdown
请帮我写一段内部通知。

背景：
公司下周五下午 2 点在 5 楼培训室举办 AI 内部分享会，各部门至少安排 1 名同事参加。

请输出：
1. 正式邮件版。
2. 企业微信群消息版，100 字以内。

限制：
不要添加原文没有的信息；缺少报名方式时标注“需确认”。
```

### 2. 总结会议记录

```markdown
请把下面的会议记录整理成会议纪要。

请输出：
1. 会议摘要，100 字以内。
2. 已确认决策。
3. 行动项表格，列为任务、负责人、截止时间、备注。
4. 风险和待确认事项。

限制：
负责人和截止时间缺失时写“需确认”，不要猜。
```

### 3. 检查邮件

```markdown
请检查下面这封邮件是否存在表达不清、语气不合适、行动要求不明确的问题。

请输出：
问题 | 修改建议 | 修改后示例

限制：
不要改变事实，不要添加承诺。
```

## 十、拿到 AI 输出后怎么检查

AI 输出后，不要直接复制使用。至少检查：

1. 是否完成了你要求的任务。
2. 是否符合指定格式。
3. 是否添加了你没有提供的信息。
4. 是否遗漏关键内容。
5. 是否有不确定但没有标注的地方。
6. 是否包含敏感信息或不合适的承诺。
7. 是否需要负责人、法务、财务、人事或业务专家审核。

## 十三、本课总结

本课最重要的变化，是把 AI 从“搜索框”变成“工作助理”。

你需要记住：

1. 模糊提问通常只能得到泛泛回答。
2. 清楚委托能让 AI 输出更接近可用结果。
3. 好 Prompt 至少包含任务、背景、资料、输出和限制。
4. 信息不足时，让 AI 先提问。
5. AI 输出仍然需要人工检查。
